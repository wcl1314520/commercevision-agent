# CommerceVision Agent

CommerceVision Agent 是面向电商视觉生产的单一编排 Agent。本文档只定义项目的统一领域语言，不记录实现方案。

## 身份

**工作区标识（Workspace ID）**:
由正则 `^[A-Za-z0-9][A-Za-z0-9._:-]{0,127}$` 定义的不透明 ASCII 租户边界标识。
系统不得修剪、折叠大小写或把非法输入规范化为另一个身份；空白、控制字符、非 ASCII 字符和
Unicode 规范化形式均为非法输入。只有完全相同的合法 Token 才表示同一工作区。
_避免使用_: 工作区名称、可规范化租户名

## 资产

**任务资产（Task Asset）**:
为一次 Workflow 上传、分析、生成或导出的输入、正文、中间结果和输出，以及承载这些内容的 Agent Checkpoint 节点数据；从 Workflow 创建起最多保存 72 小时。
_避免使用_: 临时素材、用户素材、任务数据

**基础资产（Foundation Asset）**:
经授权注册、可跨 Workflow 复用的品牌素材、Prompt 模板、LoRA、模型配置、参考素材和公开评测集；保存至管理员删除或其权利到期。
_避免使用_: 永久素材、长期任务素材、系统素材

**权利记录（Rights Record）**:
描述资产所有者、许可来源、允许用途、允许供应商、有效期和可派生条件的可审计记录。
_避免使用_: 授权备注、版权标签

**资产版本（Asset Version）**:
一次已完成上传所形成的不可变二进制内容与派生元数据快照；替换内容必须创建新版本。
_避免使用_: 文件版本、素材副本

**上传会话（Upload Session）**:
允许调用方在限定时间、对象位置、类型、大小与校验规则内向隔离区直传一个候选对象的授权。
_避免使用_: 上传链接、临时凭证

**权利到期（Rights Expiry）**:
权利记录不再允许资产继续检索、派生或发送给模型的时间边界；到期资产必须先停止使用，再完成索引和对象清理。
_避免使用_: 素材过期、自动删除时间

## 商品理解

**商品简报（ProductBrief）**:
由商品数据、任务资产和人工修订形成的结构化商品事实与视觉约束；低置信度字段必须由人工确认。
_避免使用_: 商品分析、商品 Prompt

**品牌档案（Brand Profile）**:
由有效基础资产和品牌规则组成、可版本化引用的品牌约束集合。
_避免使用_: 品牌记忆、品牌 Prompt

## 检索

**检索引用（Retrieval Citation）**:
进入 Agent 上下文的基础资产或任务资产引用，包含资产版本、权利记录和召回理由。
_避免使用_: 搜索结果、参考图链接

**检索策略（Retrieval Policy）**:
定义硬过滤、候选召回、融合、重排和上下文裁剪规则的版本化策略。
_避免使用_: 搜索配置、向量权重

## 可靠执行

**持久操作（Durable Operation）**:
以工作区、操作种类、目标身份与版本、输入哈希唯一标识的一次可恢复业务执行；它保存认领、执行、重试、对账、完成、失败和取消事实，以及不可被最新错误覆盖的 Provider Request ID、执行/对账截止时间和恢复代次。相同逻辑身份始终指向同一操作，普通消息重放不会重新打开终态；管理员显式死信重放会追加审计历史，并按失败类型只增加一次执行或对账尝试。重放保留累计尝试数并把对应最大值设为“当前计数 + 1”，因此原预算中的未使用尝试不会被重新启用。
_避免使用_: 后台任务、Celery 任务、资产 Job

**死信重放（Dead-letter Replay）**:
操作员针对一个既有死信创建的新消息投递尝试；原死信保持不变，每次重放单独记录执行人、原因、时间、序号和新事件身份。公开历史按直接子死信稳定分页，可逐层完整遍历任意深度的再次失败链。
_避免使用_: 重置死信、删除后重试
