# 产品定义与边界

| 属性 | 值 |
|---|---|
| 状态 | decision |
| 最后更新 | 2026-07-21 |

## 产品名称

**CommerceVision Agent**

副标题：Evaluation-driven Multimodal E-commerce Creative Agent。

## 产品目标

为电商美工提供一套可控制、可解释、可回放的视觉创意 Agent，将原本分散的商品分析、竞品参考、创意构思、生图、质检和返工流程自动化，同时保留关键人工决策。

项目同时承担两个目标：

1. 形成可公开访问、可真实使用的电商视觉产品。
2. 作为 Agent 应用开发求职作品，证明从 Agent 设计到生产部署的完整能力。

## 目标用户

| 用户 | 核心需求 |
|---|---|
| 电商美工 | 快速获得符合商品和品牌要求的候选图 |
| 视觉负责人 | 审核创意方案、质量和品牌一致性 |
| 商品运营 | 获取平台规范化图片包和可追溯结果 |
| 系统管理员 | 管理模型、Prompt、工具、品牌库和权限 |
| 项目评审者 | 查看架构、Trace、评测结果和生产质量 |

## 目标品类

系统架构支持多品类 Schema 和策略扩展。公开版本发布前至少覆盖：

- 美妆。
- 食品。
- 汽车配件。
- 服装。

实施时先完成两个差异明显的品类，再扩展到四类。阶段性交付只能减少已启用的品类数量，不能使用一次性代码或破坏最终架构。

## 核心能力

1. 商品图片、文本属性和品牌素材接入。
2. 多模态商品结构化理解。
3. 历史优秀素材和品牌记忆检索。
4. 结构化 Creative Plan。
5. 创意方案人工审核。
6. 多模型图片生成和编辑。
7. 文件、安全、商品一致性和平台规则评测。
8. 基于失败原因的 Repair Plan。
9. 结果人工终审和局部重做。
10. Agent Trace、Replay、Dataset 和版本对比。
11. Amazon US 图片规范化导出。
12. Web、REST API、批量任务和 Webhook。

## Agent 判定标准

系统只有同时满足以下条件才被称为 Agent：

- 能基于当前状态决定下一步骤和工具。
- 能生成受 Schema 约束的计划。
- 能读取检索记忆和工具结果。
- 能在失败后形成结构化修正策略。
- 能被人工暂停、修改和恢复。
- 决策过程可追踪、可评测、可回放。

单纯把 Prompt 发送给图片 API 不属于 Agent 核心。

## 产品边界

### 第一条完整产品线

- Amazon 主图。
- 商品场景图。
- 卖点信息图。
- 商品轻量精修。

### 长期范围

- 更多电商平台规则。
- 更细粒度局部编辑。
- 品牌视觉知识库。
- 效果数据回流。
- 企业 ERP/PIM 集成。
- 私有化部署。

### 明确不做

- 通用聊天机器人。
- 无约束浏览器自治。
- 多 Agent 社会或角色扮演。
- 视频生成。
- 完整 PSD 智能分层。
- LoRA 训练平台。
- 未经人工批准的自动商品发布。
- 抓取无授权的竞品图片。

## 非功能目标

| 维度 | 目标 |
|---|---|
| 可用性 | 公开演示控制面月可用性目标 99.9%，企业部署目标 99.95% |
| 可恢复 | 进程、Worker 或节点重启后已受理任务不丢失 |
| 可追溯 | 每张图可追溯到输入、计划、模型、Prompt、工具和审批 |
| 可评测 | 所有核心 Agent 节点有离线数据集或确定性测试 |
| 安全 | 模型不能扩展工具权限或访问任意 URL/文件 |
| 开源 | 仓库可公开，不包含商业密钥、私有素材或无许可代码 |
| 可部署 | 本地 Docker Compose 和云端 Kubernetes 均有正式路径 |

## 成功指标

### 产品指标

- 人工首轮通过率。
- 平均返工次数。
- 从提交到可审核方案的时间。
- 从方案批准到候选图完成的时间。
- 导出规则通过率。

### Agent 指标

- 结构化计划有效率。
- 非法/无效工具调用率。
- Reflection 后质量提升率。
- 回放确定性和可重现率。
- Checkpoint 恢复成功率。

### 工程指标

- API 和 Worker 测试覆盖。
- 任务丢失和重复有效生成次数。
- P95 API 时延。
- 队列等待时间。
- 模型成本和供应商错误率。
