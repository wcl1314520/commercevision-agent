# 用户与 Agent 工作流

| 属性 | 值 |
|---|---|
| 状态 | decision |
| 最后更新 | 2026-07-21 |
| 适用版本 | Product v1 |

## 主用户流程

### 1. 创建项目

用户选择商品、SKU、品牌和目标渠道，上传商品图或从商品工具读取资料。

系统完成：

- 文件真实性和安全校验。
- 商品与素材权利检查。
- 内容哈希和幂等键生成。
- 工作流和短期上下文初始化。

### 2. 商品理解

Vision Analyzer 输出结构化 `ProductBrief`：

- 商品品类和子类。
- 形状、材质、颜色、包装和接口。
- Logo、文字和保护区域。
- 核心卖点。
- 禁止改变项。
- 不确定项和人工问题。

低置信度字段必须展示给用户确认。

### 3. 记忆检索

Retriever 查询：

- 品牌规范和品牌素材。
- 同品类历史优秀素材。
- 用户明确上传的参考图。
- 历史失败案例和修正经验。

检索结果必须展示来源、相似度、业务指标和使用权，不作为隐藏上下文。

### 4. 创意规划

Planner 生成一个或多个 `CreativePlan`：

- 图片角色。
- 场景、构图、镜头、光线和色调。
- 商品保护约束。
- 选用参考素材和理由。
- 模型、工具和候选数量建议。
- 质量检查目标。
- 失败时允许的修正范围。

### 5. 人工方案审核

用户可以：

- 批准、编辑或拒绝方案。
- 锁定商品结构、Logo、文字和颜色。
- 删除不合适的参考素材。
- 选择模型、候选数量和输出尺寸。

未批准方案不能执行生图。

### 6. 工具执行

Executor 根据批准后的计划调用：

- 图片生成。
- 图片编辑或局部重绘。
- OCR 和文字检查。
- 背景处理。
- 平台规范转换。

工具由服务端注册表提供，Agent 不能动态创建工具或传入任意文件路径。

### 7. 自动评测

Evaluator 产生 `EvaluationReport`：

- 文件完整性。
- 内容安全。
- 商品主体一致性。
- Logo 和包装文字。
- 品牌视觉一致性。
- 创意计划遵循度。
- Amazon 规则。

### 8. 反思与修正

当结果未达到阈值时，Reflector 生成 `RepairPlan`：

- 失败证据。
- 根因分类。
- 是否可自动修复。
- 应修改的 Prompt、工具、参考图或模型。
- 不允许改变的内容。

自动修正有最大次数和成本边界。达到边界后必须人工处理。

### 9. 人工终审

用户选择通过、拒绝或局部重做。终审反馈以结构化原因保存，进入评测数据集候选区，但不会自动污染线上 Prompt。

### 10. 导出

系统生成：

- 审批后的原始图片。
- Amazon 角色命名图片。
- `manifest.json`。
- `validation-report.json`。
- Trace 和版本摘要。

## 异常流程

| 场景 | 系统行为 |
|---|---|
| 模型超时 | 有界重试，必要时切换兼容端点 |
| 内容策略拒绝 | 不跨供应商规避，转人工处理 |
| 供应商结果未知 | 根据 request/task ID 对账，不盲目重发 |
| 用户长期未审核 | 提醒并按数据生命周期到期 |
| 检索结果无授权 | 从上下文剔除并记录审计 |
| 自动评测冲突 | 保留各 Evaluator 原始结果，交人工裁决 |
| 工作进程退出 | 从 MySQL Checkpoint 和任务状态恢复 |
| 重复 Webhook | 根据事件 ID 和幂等键忽略重复 |

## 公开演示流程

公开环境只提供经过授权或合成的样本：

1. 选择公开商品。
2. 查看 Agent 分析和检索过程。
3. 修改并批准 Creative Plan。
4. 观察生成、评测和修正 Trace。
5. 下载带水印或限制分辨率的演示结果。

公开环境必须设置用户配额、供应商预算和内容安全限制。
