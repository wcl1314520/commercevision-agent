# ADR-004：单一 Agent 与 LangGraph

| 属性 | 值 |
|---|---|
| 状态 | accepted |
| 日期 | 2026-07-21 |

## 背景

多 Agent 会增加上下文同步、责任边界、评测和恢复复杂度，但当前业务可以由一个受控编排 Agent 完成。

## 决策

- 使用单一 LangGraph Agent。
- 分析、检索、规划、执行、评测和反思是 Graph 节点，不是多个自治角色。
- 使用自定义 MySQL Checkpointer。
- 确定性领域规则不交给 LLM 决定。

## 后果

- 状态和责任更清晰。
- Human-in-the-loop 和回放更容易实现。
- 未来只有出现独立目标、独立记忆和独立评测需求时才引入额外 Agent。

## 验证

- Graph 可以完整表达主流程和异常流程。
- 无需 Agent 间自然语言协商。
- 每个节点可单测和独立评测。
